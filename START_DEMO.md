# PrepMeAI - Demo Startup Guide

## Quick Start (For Tomorrow's Demo)

### Prerequisites
- Python 3.10+ installed
- Node.js 18+ installed
- Groq API key (already in `.env`)

### Step 1: Install Backend Dependencies
```bash
cd backend
pip install -r requirements.txt
```

**Expected time**: 2-3 minutes

### Step 2: Start Backend Server
```bash
# From backend directory
uvicorn main:app --reload --port 8000
```

**Verify**: Open http://localhost:8000/docs - you should see FastAPI Swagger UI

### Step 3: Install Frontend Dependencies (if not done)
```bash
cd frontend
npm install
```

### Step 4: Start Frontend
```bash
# From frontend directory
npm run dev
```

**Verify**: Open http://localhost:3000

## Key Points to Emphasize

### 1. RAG Implementation
**Explain**: "We use Retrieval-Augmented Generation to ensure accuracy"
- PDF is chunked into paragraphs
- Each chunk is embedded using sentence-transformers
- Student questions are matched using hybrid search (70% semantic + 30% keyword)
- Top 5 relevant chunks are sent to Groq LLM
- LLM generates answer ONLY from provided context
- **No hallucination** - all answers are grounded in NCERT textbook

### 2. Vector Database
**Explain**: "We use in-memory vector embeddings for fast retrieval"
- Sentence-transformers creates 384-dimensional vectors
- Cosine similarity finds semantically similar content
- BM25 adds keyword matching for rare terms
- Currently in-memory (NumPy), can scale to Pinecone/Weaviate

### 3. Granular Difficulty
**Explain**: "5 distinct difficulty classes with specific rubrics"
- Each level has different assessment criteria
- Very Easy: Just recall the definition
- Medium: Need explanation + example
- Very Hard: Synthesize multiple concepts
- Rubric is sent to LLM for consistent grading

### 4. Dynamic Feedback
**Explain**: "Assessment is not template-based, it's generated by LLM"
- LLM compares student answer vs. textbook content
- Identifies specific points covered and missed
- Provides actionable improvement suggestions
- Model answer is extracted from PDF context

## Troubleshooting

### Backend won't start
```bash
# Check if port 8000 is in use
netstat -ano | findstr :8000

# Kill the process if needed
taskkill /PID <process_id> /F

# Restart backend
uvicorn main:app --reload --port 8000
```

### Frontend shows "Failed to fetch"
- Ensure backend is running on port 8000
- Check browser console for CORS errors
- Verify API_BASE in frontend code is "http://localhost:8000"

### PDF not found error
- Ensure `ncert_science_8.pdf` is in project root
- Check `backend/.env` has correct `PDF_PATH=../ncert_science_8.pdf`

### Voice input not working
- Use Chrome/Edge (Safari has limited support)
- Allow microphone permissions when prompted
- Check browser console for errors

## Technical Architecture Summary

```
Student Question
    ↓
[Frontend] Voice/Text Input
    ↓
[API] POST /api/quiz/assess or /api/tutor/ask
    ↓
[RAG Service] 
    1. Embed query (sentence-transformers)
    2. Hybrid search (70% dense + 30% BM25)
    3. Retrieve top-5 chunks from PDF
    ↓
[Groq LLM] 
    - System prompt: "Use only provided context"
    - Context: Retrieved PDF chunks
    - Generate: Answer/Assessment
    ↓
[Frontend] Display with sources
    ↓
[TTS] Read aloud (optional)
```

## Questions Mentors Might Ask

**Q: How do you prevent hallucination?**
A: System prompt explicitly forbids external knowledge, only retrieved PDF chunks are in context, and we show source page numbers for verification.

**Q: What if the PDF doesn't have the answer?**
A: RAG service returns "I couldn't find relevant information in the textbook" if retrieval score is too low.

**Q: How does voice assessment work?**
A: Web Speech API transcribes voice to text, then text is assessed same as typed answers using RAG.

**Q: Can this scale to multiple textbooks?**
A: Yes, we'd need to add document metadata to chunks and filter by subject/grade during retrieval.

**Q: Why not use a vector database?**
A: For single PDF (~300 chunks), in-memory is faster. For production with multiple books, we'd use Pinecone or Weaviate.

**Q: How accurate is the assessment?**
A: LLM compares against actual textbook content, so accuracy depends on Groq's reasoning ability. We use Llama 3.3 70B for high quality.
